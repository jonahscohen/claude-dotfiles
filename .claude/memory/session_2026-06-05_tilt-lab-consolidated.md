# tilt-lab - consolidated

Consolidation of 39 `type: project` tilt-lab session beats spanning 2026-05-29 to 2026-06-04 (collaborator: Jonah). tilt-lab is a sidecoach-owned, locally-served visual-effects playground in claude-dotfiles: browse a catalog of ~25 generative/shader effects, layer compatible ones, tune params live, and export a paste-anywhere embed that renders the exact previewed stack in any project. This canonical carries forward every durable fact and decision at recall level; the 39 originals remain on disk (superseded, never deleted) and are grep-able for deep detail. The standing beats that govern this work stay individually visible and were NOT consolidated: `feedback_tilt_lab_fidelity_mandate.md` (1:1 fidelity mandate), `session_2026-05-29_tilt-lab-asset-gap-FAILURE.md` + `_recon-team.md` (feedback), and the decision beats `_design-direction.md`, `_pointer-contract-gap.md`, `_recon-synthesis.md`, `_slider-accent.md`.

## Concept, scope, and architecture
- tilt-lab (named by Jonah, was working title "shader-lab"), inspired by basement.studio's shader-lab. Lives in claude-dotfiles, owned by sidecoach, launched on demand into a cmux pane / browser tab, ALWAYS served locally. (brainstorm)
- 4 locked foundational decisions (AskUserQuestion): (1) home = claude-dotfiles, sidecoach-owned, local-only; (2) output format = framework-agnostic Web Components (`<tilt-{id}>`, `<tilt-stack>`); (3) sourcing posture = verbatim everything, personal-use, attribution/license/redistribution stored as per-effect metadata (not hard-enforced in v1); (4) app stack = Vite + React + TS shell, effects stay framework-free web components. Package dir `<project>/tilt-assets/<stack-name>/`. (brainstorm)
- Runtime contract: `Effect { init, frame, resize, setParam, dispose }`. Manifest carries id/name/category/layerRole/params/requiredAssets/origin/license/attribution/redistribution/tags. Generic manifest-driven web-component wrapper owns canvas mount, RAF, ResizeObserver, prefers-reduced-motion, dispose. (brainstorm)
- Layering model: 4 layer roles - background (exclusive max 1), midground (stackable), pointer (stackable), post (exclusive max 1, composited last). Valid order bg -> mid -> pointer -> post. Playground greys out incompatible adds with a reason. The post layer must sample the composited output beneath (the known hard part). (brainstorm)
- Repo dir: `tilt-lab/{app (Vite+React), runtime (effect contract + web-component bundles), runtime/effects (per-effect + manifest.json), runtime/lib, server, dist}`. Top-level `shaders/` is unrelated ghostty terminal shaders. (brainstorm, foundation-plan)

## Compositor render bug (systemic, found by visual verification) - FIXED
- Symptom: clicking an effect added the layer + param controls but the CENTER PREVIEW stayed BLACK. (compositor-bug)
- Two-part rework first: the Plan-1 Compositor never sized its canvas or called effect.resize(), and shared ONE canvas across all layers (a canvas holds only one context kind). Reworked to per-layer OWN element stacked in the host (position:absolute; inset:0; 100%; mix-blend-mode per layer; z-index by order), ResizeObserver driving resize() for all layers, PointerTracker on the host root forwarding onPointer. (compositor-bug, ui-exec)
- ROOT CAUSE of the persistent blackness (found via instrument + cmux console, a debugging-protocol win over theorizing): the compositor's own one-liner `if (!this.root.style.position) this.root.style.position='relative'` clobbered PreviewCanvas's `.preview-canvas` (which fills its parent via `position:absolute; inset:0`), collapsing the host to ~0 height -> canvas h=1 (2px) -> invisible. Hit EVERY layer (Canvas2D + WebGL). FIX: only set relative when computed position is `static`/empty (never clobber absolute/relative/fixed). After fix lava-lamp (OGL) + aurora (raw WebGL2) render full-screen, visually confirmed. Earlier hypotheses (effects create own canvas; double-getContext attribute mismatch) were red herrings. (compositor-bug, ui-exec)
