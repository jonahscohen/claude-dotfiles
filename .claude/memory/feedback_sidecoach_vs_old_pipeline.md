The claude-dotfiles reference landing page (port 8766) was built with the old pipeline, not Sidecoach. Despite being imperfect, it was "miles better" than what Sidecoach produced.

**Why:** Sidecoach's elaborate validator framework (159 rules, 36 flows, extended domain validation) did not translate to better actual design output. The old, simpler approach was more effective.

**How to apply:** Recognize that orchestration complexity doesn't guarantee quality. Measure deliverable output, not architectural sophistication. When Sidecoach is used for real work in future, validate against concrete outputs first before trusting the system.
