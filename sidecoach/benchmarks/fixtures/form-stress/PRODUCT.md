# PRODUCT.md

## Register

**Product**

## Primary Users

Government and enterprise users completing a multi-step intake form. Audience skews older, screen-reader users are over-represented, and the form must work on a ten-year-old desktop browser as well as a current iPhone.

## Brand Personality

Clear, calm, deliberately boring. The form should feel like a serious public-service tool, not a marketing funnel. No copy that reads like a startup growth team wrote it.

## Anti-References

- Multi-column forms with required-field stars only
- Inline validation that fires on every keystroke
- Floating-label inputs without a fallback label
- Form errors at the top of the page only
- Submit buttons that look like links

## Strategic Principles

- WCAG 2.1 AA is the floor, not the ceiling
- Every input has a visible label, programmatically associated
- Errors appear inline AND in a summary, in plain language
- Tab order matches visual order
- All interactive targets are 44x44 minimum
- Color is never the only signal for state
